Accessibility Statement

Making my website usable for everyone

Semantic HTML

I use semantic elements like <header>, <nav>, <main>, and <footer> to help screen readers understand the page structure.

Alt Text for Images

All images include descriptive alt attributes so visually impaired users understand the content.

Color Contrast

I chose color combinations with sufficient contrast to ensure text is readable for users with low vision.

Keyboard Navigation

All interactive elements can be accessed using the Tab key, making the site usable for people who cannot use a mouse.

Responsive Design

The website works on all screen sizes, from desktop to mobile phones, using media queries and flexible layouts.

Clear Headings

I use proper heading hierarchy (h1 to h6) to organize content logically for screen reader users.

Why Accessibility Matters

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with websites. According to the World Health Organization, over 1 billion people live with some form of disability. Making my websites accessible means I am not excluding anyone from accessing information and services online.
