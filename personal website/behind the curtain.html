Behind the Curtain

How I built this website

Project Setup

I started by creating a new folder in VS Code and setting up the basic file structure. I created separate files for HTML, CSS, and JavaScript to keep my code organized and maintainable.

portfolio/
├── index.html
├── portfolio.html
├── design-thinking.html
├── global-goal.html
├── accessibility.html
├── behind-the-curtain.html
├── styles.css
└── script.js

Challenges I Faced

The most challenging part was making the navigation bar work on both desktop and mobile. I used a combination of Flexbox for desktop and JavaScript with media queries for the mobile menu. Another challenge was getting the color scheme to look professional while maintaining good contrast for accessibility.

Using Developer Tools

I used Chrome DevTools extensively to test my layouts, debug CSS issues, and check responsive behavior. The inspector helped me see exactly which styles were applied and where conflicts were happening.

What I Learned

  • Planning before coding saves time and prevents major redesigns
  • CSS Grid and Flexbox make responsive layouts much easier
  • Accessibility should be considered from the start, not added at the end
  • Getting feedback early and often improves the final product
  • JavaScript can make static pages feel alive and interactive
