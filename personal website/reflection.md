When I started this course I knew absolutely nothing about web design. The thought of creating a website from scratch was daunting. I was nervous about learning to code, all the HTML tags, CSS properties and JavaScript syntax. But now when I look at my finished portfolio website I feel proud of how much I’ve grown.

Making my website responsive was one of the biggest challenges I faced. Initially, my nav bar was fine on a computer screen, but broken on mobile phones. So I learned to use CSS media queries to fix this, and now my site works on any device. Another problem was getting my images to show up correctly. I had to learn how to sort files and use proper file. Another challenge was to get my images to show up properly. I learned how to organize files and use the correct file paths.

I had a hard time with JavaScript myself when I started out. The buttons responding to the click and the typing effect took a lot of tries. But each time things failed I learned something new. I used the developer tools in the browser to quickly find and fix errors.

The design thinking process was very useful. I learned to lead with empathy – understanding what users need – before writing any code. This gave my designs more meaning. I chose colors, content that reflects my personality, not just what looks trendy.

Accessibility was another key lesson. I added alt text to all images, used semantic HTML and ensured my color contrast was readable. I want my website to be usable by everyone, including those who use screen readers or can't use a mouse.

Given more time I would add more features in Javascript like a dark mode toggle or a simple game. I would also like to understand backend development more so I can add user accounts or save data.

Overall this class taught me that web design is not just about code, but creativity, problem solving and empathy. I look forward to learning and building more websites in the days to come.
